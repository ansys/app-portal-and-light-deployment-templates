Configure platform deployment#

Note

This section is optional and only required if you need to change the configuration of the platform.

This section assumes that you have already deployed the platform as indicated in Platform deployment.

The platform supports configurations through the use of below environment variables. These environment variables are defined in the .env file located in the root directory of the platform.

Note

If the platform is already running, before making any changes in the .env file, make sure to stop the platform services by running the following command:

Open a Windows powershell terminal with administrator privilege and go to the directory deployment_scripts:

  • powershell
cd .\Ansys_App_Portal_and_Deployment_Templates\deployment_scripts

Run the following command to stop the platform, including containerized services and desktop processes:

  • powershell
.\Stop-Platform.ps1

Variable

Description

SOLUTIONS_DIRECTORY_PATH
Path to the directory where the solutions are stored.
Default to C:\Users\<user_name>\AppData\Roaming\Ansys\solutions

ANSYS_242_INSTALLATION_PATH

Path to the Ansys optiSLang installation directory. Default to C:\Program Files\Ansys Inc\v242

OSL_SAF_PACKAGES_DIRECTORY_PATH
Path to the directory where the OSL SAF packages are stored.
Default to C:\\Program Files\\ANSYS Inc\\v242\\optiSLang\\lib\\python3.10\\Lib\\saf-packages

PIM_PORT

Port used by the instance management system. Default to 8443

KC_PORT

Port used by the Keycloak server. Default to 8011

KC_ADMIN_USER

Keycloak admin user name. Default to admin

KC_ADMIN_PASSWORD

Keycloak admin password. Default to admin

OP_EMAIL_DOMAIN

Email domain used during the user registration process. Default to example.com

PLATFORM_DEPLOYMENT_LOGS_DIR
Path to the directory where the platform deployment logs are stored.
Default to C:\Users\<user_name>\AppData\Roaming\Ansys_solutions_platform_logs

Attention

If users are already registered in the platform, changing the OP_EMAIL_DOMAIN environment variable will affect the existing users. Existing users will no longer be able to log in with their old email domain. They will need to re-register using the new email domain.

After making changes in the .env file, start the platform services by running the following commands.

Open a Windows powershell terminal with administrator privilege and go to the directory deployment_scripts:

  • powershell
cd .\Ansys_App_Portal_and_Deployment_Templates\deployment_scripts

Run the following command to start the platform:

  • powershell
.\Start-Platform.ps1
