Welcome to the **Ansys App Portal and Deployment Templates**
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~


.. only:: external

   .. list-table::
      :stub-columns: 1
      :widths: 25 75

      * - Project name:
        - ``Ansys App Portal and Deployment Templates``
      * - Package name:
        - ``ansys-app-portal-and-deployment-templates``
      * - Supported Python versions:
        - 3.9 to 3.11
      * - Supported operating systems
        - Windows 10, Windows 11, and Windows Server 2022


The :program:`Ansys App Portal and Deployment Templates` provide all the containerized services needed to leverage single-node deployments for the execution of solution app instances. This allows you to deploy an **Ansys optiSLang®** web app (AWA) on the same node where optiSLang is installed.

.. note::
   These templates are compatible only with web apps created with either **Ansys optiSLang® 2024 R2** release or **Ansys optiSLang® 2024 R2 SP2** service pack.

   Apps created from **Ansys optiSLang® 2024 R2** and **Ansys optiSLang® 2024 R2 SP2** service pack cannot be deployed together. 
   For more information, see the :ref:`known_limitations`

Services included:
~~~~~~~~~~~~~~~~~~

Ansys App Portal:
 User interface for managing solution apps and the projects created for them.

Product Instance Manager (PIM):
 Manages the starting and stopping of the **Ansys optiSLang®** product.

Authentication services:
 Used to authenticate users and provide access to platform services.

Databases:
 Used to store and manage data in the Ansys App Portal and in each solution app.

API Gateway:
 Provides a single entry point for managing, routing, and securing API requests between clients and backend services.

App service:
 Used to manage containerized applications in the platform.

App Adapter service:
 Used to include the necessary Dockerfiles and Docker Compose files for running applications.

Observability services:
  Used to analyze and understand system behavior through metrics, logs, and traces.
